## Points d'attention (pas des alertes — a verifier en Session 2)

1. **`orders` et `order_items` ont des policies "Anon peut lire les commandes" et "Anon peut lire les items"**.
   A confirmer : ces policies sont-elles destinees au suivi public via token (page /suivi) ?
   Si oui, elles devraient etre limitees par `WHERE id = current_setting('order.token')` ou equivalent.
   Sinon, elles permettent a n'importe quel anon de lire toutes les commandes.

2. **`delivery_tokens` a une policy "Update token public (w)"**.
   A confirmer : cette policy doit etre limitee a `WHERE used = false` pour empecher la reutilisation.

3. **`leads` a une policy "Autoriser lecture leads pour admin (r)"**.
   A verifier : la condition de la policy filtre bien par role admin et pas anon.

Pour voir le detail de chaque policy (`USING` clause complete) :

```bash
supabase db query --linked --agent=no "SELECT polname, pg_get_expr(polqual, polrelid) AS using_clause FROM pg_policy p JOIN pg_class c ON p.polrelid = c.oid WHERE c.relnamespace = (SELECT oid FROM pg_namespace WHERE nspname = 'public') AND c.relname IN ('orders', 'order_items', 'delivery_tokens', 'leads') ORDER BY c.relname, polname;"
```
